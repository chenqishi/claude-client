## ADDED Requirements

### Requirement: Help card contains Skill list button
The help card generated by `generateHelpCard()` SHALL include a "Skill 列表" button in the action section after the permission mode section. The button's `value.command` SHALL be `/skills`.

#### Scenario: User views help card
- **WHEN** user sends `/help` or `?` to the bot
- **THEN** the help card SHALL contain a button labeled "Skill 列表" with `value.command` set to `/skills`

#### Scenario: User clicks Skill list button
- **WHEN** user clicks the "Skill 列表" button in the help card
- **THEN** the bot SHALL receive a callback with `command: '/skills'` and display the skill list card

### Requirement: Skill list command recognition
`detectSpecialCommand()` SHALL recognize `/skills` as a command with type `skills`.

#### Scenario: User types /skills
- **WHEN** user sends `/skills` as a message
- **THEN** `detectSpecialCommand()` SHALL return `{ type: 'skills', args: '' }`

### Requirement: Skill list card generation
The system SHALL provide a `generateSkillListCard()` function that builds a Feishu interactive card listing all available skills grouped by category. Each skill SHALL be displayed as a button with the skill's display name. Each button's `value.command` SHALL be `/skill-copy <skill-id>`.

The card SHALL include a "返回帮助" button with `value.command` set to `/help`.

#### Scenario: Skill list card structure
- **WHEN** `generateSkillListCard()` is called
- **THEN** the returned card SHALL have header title "Skill 列表" and template "blue"
- **AND** the card SHALL contain category sections with skill buttons organized under each category
- **AND** each skill button SHALL have `value.command` set to `/skill-copy <skill-id>`
- **AND** the card SHALL end with a "返回帮助" button

#### Scenario: Skill list card shows categories
- **WHEN** `generateSkillListCard()` is called
- **THEN** the card SHALL display at minimum the following categories: "工作流", "代码质量", "调试分析"
- **AND** each category SHALL have a `lark_md` div section header followed by skill buttons

### Requirement: Skill copy command recognition
`detectSpecialCommand()` SHALL recognize `/skill-copy` prefix as a command with type `skill-copy`. The `args` field SHALL contain the skill identifier after the prefix.

#### Scenario: User triggers skill copy via button
- **WHEN** button click callback sends `/skill-copy autopilot`
- **THEN** `detectSpecialCommand()` SHALL return `{ type: 'skill-copy', args: 'autopilot' }`

#### Scenario: Skill not found
- **WHEN** `detectSpecialCommand()` receives `/skill-copy unknown-skill`
- **THEN** it SHALL return `{ type: 'skill-copy', args: 'unknown-skill' }` (validation happens in handler)

### Requirement: Skill copy handler sends command template
When the `skill-copy` command is handled, the bot SHALL look up the skill by its identifier in the skill data. If found, the bot SHALL send a text message containing the full command template with usage instructions. If not found, the bot SHALL send a "skill 未找到" error message.

#### Scenario: Successful skill copy
- **WHEN** `handleSpecialCommand` processes `skill-copy` with args `autopilot`
- **THEN** the bot SHALL send a text message containing `/oh-my-claudecode:autopilot <任务描述>`
- **AND** the message SHALL include instructions for the user to long-press copy and paste

#### Scenario: Skill not found
- **WHEN** `handleSpecialCommand` processes `skill-copy` with args `nonexistent`
- **THEN** the bot SHALL send a text message indicating the skill was not found

### Requirement: Skill data structure
The system SHALL define a `SKILL_CATEGORIES` constant containing an array of category objects. Each category SHALL have a `name` (display name), and a `skills` array. Each skill SHALL have: `id` (unique identifier), `name` (display name), `command` (full command template), and `description` (brief description).

#### Scenario: Skill data is well-formed
- **WHEN** `SKILL_CATEGORIES` is defined
- **THEN** each skill SHALL have all four fields: `id`, `name`, `command`, `description`
- **AND** each `id` SHALL be unique across all categories
- **AND** each `command` SHALL start with `/`
