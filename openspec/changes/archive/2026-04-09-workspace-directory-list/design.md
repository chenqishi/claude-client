## Context

Claude Client 是一个飞书机器人，桥接飞书消息与本地 Claude Code CLI。当前用户切换工作目录只能通过 `/cd <path>` 手动输入路径，或点击 Help 卡片中的"最近访问目录"按钮。对于在固定根目录（如 `D:\code`）下管理多个项目的开发者，缺少一个直观的方式浏览和选择项目。

现有架构：
- `config.ts` 通过 dotenv 加载环境变量，提供单例 `AppConfig`
- `formatter.ts` 中 `generateHelpCard()` 生成飞书交互卡片，包含可点击按钮
- `handler.ts` 中 `handleCardAction()` 处理卡片按钮点击，提取 `command` 字段路由到命令处理
- `app.ts` 中 help 分支处理 `?`/`/help` 命令

## Goals / Non-Goals

**Goals:**
- 通过环境变量配置一个工作目录根路径
- 启动时读取该路径下的子文件夹列表
- Help 卡片增加"目录列表"入口按钮
- 目录列表卡片风格与 Help 卡片一致，每个子目录为可点击按钮
- 点击子目录按钮执行 `/cd <path>` 切换目录

**Non-Goals:**
- 不实现目录的增删改操作（只读浏览）
- 不实现目录搜索或过滤功能
- 不实现多级目录嵌套浏览（只展示一级子文件夹）
- 不改变现有的 `/cd` 命令行为和"最近访问目录"功能

## Decisions

### 1. 使用环境变量 `WORKSPACE_DIR` 配置工作目录

**选择**: 在 `.env` 中新增 `WORKSPACE_DIR` 环境变量
**理由**: 与现有配置模式一致（`CLAUDE_CODE_PATH`、`SESSION_TIMEOUT_MINUTES` 等都用环境变量）
**备选**: 在 `settings.json` 中动态配置 — 增加了复杂度，且工作目录根路径通常不需要动态修改

### 2. 子文件夹列表在卡片请求时动态读取

**选择**: 每次用户点击"目录列表"时，实时读取 `WORKSPACE_DIR` 下的子文件夹
**理由**: 避免启动时缓存导致新增项目不显示；文件夹数量通常不多（<50），文件系统读取开销可忽略
**备选**: 启动时缓存列表，提供刷新按钮 — 增加状态管理复杂度，收益不大

### 3. 目录列表作为独立卡片发送

**选择**: 点击 Help 卡片中的"目录列表"按钮后，发送一张新的独立卡片
**理由**: 与现有模式一致（Help 卡片按钮触发新消息）；避免 Help 卡片过长
**备选**: 在 Help 卡片中内嵌展开区域 — 飞书卡片支持有限，交互复杂

### 4. 复用现有 card action 处理机制

**选择**: 目录列表卡片的按钮 value 使用 `{ command: "/cd <path>" }` 格式
**理由**: 完全复用 `handler.ts` 中现有的 `handleCardAction()` 逻辑，无需修改交互处理代码

## Risks / Trade-offs

- **[WORKSPACE_DIR 未配置]** → 用户不配置时，Help 卡片不显示"目录列表"按钮，功能静默降级
- **[工作目录路径无效或无权限]** → 点击时返回错误提示，不影响其他功能
- **[子文件夹数量过多]** → 全部展示，卡片内容可滚动浏览
- **[路径中包含特殊字符]** → 使用 `path.resolve()` 标准化路径，飞书按钮 value 支持 JSON 字符串
